# 《操作系统》课程设计报告模板

## 1. 项目概述

本项目围绕操作系统核心机制，使用 C++17 实现处理机调度、内存管理、进程同步与简易文件系统四个基础模块，并在扩展部分完成调度性能测试与优化分析。程序采用菜单式交互，支持默认样例演示、动态输入和自动化性能测试。

## 2. 开发环境

- 操作系统：Linux / WSL / openEuler
- 编译器：g++
- 构建工具：make
- 线程库：C++ 标准线程库 `std::thread`

## 3. 处理机调度

### 3.1 实现算法

- FCFS：按照进程到达时间依次调度。
- SJF：在已到达进程中选择服务时间最短者。
- RR：按照时间片轮转执行，记录每次运行区间。
- 优先级调度：数值越小优先级越高。

### 3.2 输出结果

程序输出运行顺序、完成时间、等待时间、周转时间、平均等待时间和平均周转时间。

### 3.3 算法分析

FCFS 简单公平但可能产生护航效应；SJF 平均等待时间较低但可能造成长作业饥饿；RR 响应快，适合分时系统；优先级调度能体现任务重要性，但低优先级任务可能长期等待。

## 4. 内存管理

### 4.1 动态分区

实现首次适应 FF 和最佳适应 BF，展示分配、回收、空闲分区合并过程。

### 4.2 页面置换

实现 FIFO 和 LRU 页面置换，统计缺页次数和缺页率。

### 4.3 实验分析

FF 查找速度快，但低地址碎片较多；BF 尽量减少单次剩余空间，但容易产生小碎片。FIFO 简单但可能淘汰热点页面；LRU 利用局部性原理，通常表现更稳定。

## 5. 进程同步与并发控制

### 5.1 生产者-消费者

使用 `std::condition_variable` 控制缓冲区空/满状态，使用 `std::mutex` 保护缓冲区访问。

### 5.2 读者-写者

多个读者可并发读取，写者需要独占访问共享数据。

### 5.3 哲学家进餐

使用 `std::lock` 同时申请两把筷子的互斥锁，避免线程间死锁。

## 6. 文件系统

### 6.1 设计思想

使用目录表保存文件名、文件大小和占用块号；使用位示图管理空闲磁盘块。

### 6.2 基本操作

实现文件创建、写入、读取、删除、目录查看和空闲空间查看。

## 8. 扩展提升部分：调度性能测试与优化分析

### 8.1 选题方向

本项目扩展方向选择“调度与性能优化”。该方向与基础部分的处理机调度模块联系紧密，不依赖内核编译、驱动开发或特殊硬件平台，可直接运行得到实验结果，适合课程设计报告展示。

### 8.2 实现内容

扩展模块自动构造 balanced 和 bursty 两组进程负载，对 FCFS、SJF、RR、Priority 和 HRRN 算法进行统一测试。其中 HRRN 为最高响应比优先算法，响应比计算公式为：

```text
响应比 = (等待时间 + 服务时间) / 服务时间
```

HRRN 会随着等待时间增长提高长作业优先级，因此能够在保留短作业优先优势的同时，缓解长作业饥饿问题。

### 8.3 评价指标

- 平均等待时间：反映进程在就绪队列中等待的平均时长。
- 平均周转时间：反映进程从到达到完成的平均总耗时。
- 平均响应时间：反映进程从到达到第一次被调度的等待时间。
- 上下文切换次数：反映调度频繁程度和潜在切换开销。
- 吞吐率：单位时间内完成的进程数量。

### 8.4 实验结果

运行菜单中的“扩展提升：调度性能测试与优化分析”，程序会在控制台输出性能对比表，并生成 `docs/scheduler_benchmark.csv`。报告中可粘贴控制台表格截图，也可以将 CSV 内容整理为 Word 表格。

### 8.5 结果分析

RR 通常具有更好的响应性，但上下文切换次数较多；SJF 在短作业较多时平均等待时间较低，但可能导致长作业等待过久；HRRN 通过响应比动态提升等待时间较长的进程优先级，在公平性和效率之间取得更平衡的效果。
